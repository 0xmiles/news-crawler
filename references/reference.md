# Blog Writing Style Reference (for Claude Code SubAgent)


이 문서는 Claude Code SubAgent가 기술 블로그 글을 작성할 때 참고해야 할 **글쓰기 스타일, 톤, 구조, 사고 방식**을 정의합니다.  
목표는 *전문성과 실용성을 동시에 갖춘 개발자 블로그*를 일관된 품질로 생산하는 것입니다.

---

## 1. 글의 목적과 독자

### 1.1 주요 독자
- 현업 개발자 (주니어 ~ 시니어)
- AI 도구, 자동화, 생산성 향상에 관심 있는 엔지니어
- Claude Code / AI Agent를 실무에 활용하려는 독자

### 1.2 글의 목적
- 새로운 개념을 “처음 보는 사람도 이해할 수 있게” 설명한다
- 단순 기능 나열이 아니라 **왜 유용한지, 언제 쓰는지**를 전달한다
- 읽고 나서 바로 실무에 적용할 수 있는 힌트를 제공한다

---

## 2. 전체 톤 & 스타일 가이드

### 2.1 기본 톤
- **전문적이지만 딱딱하지 않게**
- 강의체 ❌ / 경험 공유 + 정리하는 느낌 ⭕
- 독자에게 말을 거는 듯한 문장 구성

### 2.2 문체 원칙
- 능동태 위주로 작성
- 너무 학술적인 표현은 피하고, 실무 언어를 사용
- 불필요한 미사여구는 제거
- “~입니다” 체를 기본으로 하되, 필요하면 가볍게 풀어쓴 문장 허용

예시:
- ❌ 본 문서에서는 Claude Code의 구조적 특성을 고찰한다
- ⭕ Claude Code를 쓰다 보면, 이런 구조가 왜 필요한지 자연스럽게 느끼게 됩니다

---

## 3. 글의 구조 패턴

### 3.1 기본 흐름
1. 도입부 (Hook)
   - 질문, 문제 제기, 트렌드 언급
   - 독자가 “이거 내 얘긴데?”라고 느끼게 만들기

2. 배경 설명
   - 왜 이 주제가 나왔는지
   - 기존 방식의 한계 또는 불편함

3. 핵심 개념 설명
   - 개념 정의
   - Claude Code 관점에서의 의미
   - 다른 도구와의 차이점

4. 실전 예시
   - 설정 파일
   - 코드 블록
   - 실제 사용 시나리오

5. 정리 및 다음 단계
   - 언제 쓰면 좋은지
   - 처음 시작하는 방법

---

## 4. 독자와의 대화 방식

### 4.1 직접 호칭
- 여러분
- 개발하다 보면
- 한 번쯤 이런 경험 있으실 겁니다

### 4.2 자주 사용하는 연결 문장
- 여기서 한 가지 짚고 넘어가야 할 점이 있습니다
- 이 부분이 은근히 중요합니다
- 처음 보면 헷갈릴 수 있는데요
- 개인적으로 가장 자주 쓰는 방식은…

### 4.3 수사적 질문 활용
- 굳이 이렇게까지 해야 할까요?
- 그럼 언제 SubAgent를 써야 할까요?
- 이걸 매번 수동으로 해야 한다면 어떨까요?

---

## 5. 코드 & 예시 작성 원칙

### 5.1 코드 블록 사용 기준
- 설명 없이 코드만 던지지 않는다
- 코드 앞에는 반드시 **맥락 설명 문장**을 둔다

### 5.2 예시는 현실적인 수준으로
- Hello World 수준은 피한다
- 실제 프로젝트에서 쓸 법한 예제를 사용한다

---

## 6. 주장과 의견 표현 방식

### 6.1 개인 경험 기반
- 공식 문서 인용보다는 실제 사용 경험을 우선한다

### 6.2 단정 대신 여지 남기기
- “정답이다” 보다는 “잘 맞았다”, “유용했다” 표현 사용

---

## 7. 마무리 스타일

### 7.1 결론은 가볍게
- 요약 bullet point
- 다음 글 예고
- 독자 행동 유도

### 7.2 교훈적으로 끝내지 않기
- 강의자료처럼 마무리하지 않는다

---

## 8. 피해야 할 것들
- 과도한 영어 남발
- 공식 문서 문장 그대로 사용
- 기능 목록 나열로 끝내기
- AI 찬양 일변도 서술

---

## 핵심 요약
- 정리된 경험 공유를 목표로 한다
- 독자를 같은 개발자로 전제한다
- Claude Code를 함께 일하는 환경으로 설명한다
- 읽고 나면 바로 써보고 싶게 만든다
